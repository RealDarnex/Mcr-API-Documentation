Usage

EntityAPI.kill(Entity entity);

SpigotAPI Equivalent

entity.setHealth(0);

Description

The kill() method terminates an entity by setting its health to 0. Internally, this mirrors the behavior of calling setHealth(0) on a LivingEntity within the Spigot API. As a result, the entity dies in the same way as it would through normal gameplay mechanics.

Details

This method only applies to entities that extend LivingEntity (e.g., players, mobs, animals). For all other entity types (such as dropped items, projectiles, armor stands, or decorative entities), the method will not execute successfully and will instead trigger an error code.

When successful, the method triggers the appropriate Spigot events, such as EntityDeathEvent for mobs or PlayerDeathEvent for players. This ensures compatibility with plugins or systems that listen for entity death events.

Available Error Codes

Error 179: The entity does not exist (null).

Error 181: The entity is not a LivingEntity.
